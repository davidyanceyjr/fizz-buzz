# phase-oriented-c-review

Purpose: Ensure C code follows phase-oriented structure.

Procedure:
1. Identify logical phases of execution.
2. Check nesting depth and readability.
3. Verify centralized cleanup paths.
4. Ensure explicit resource ownership.
5. Confirm error paths are consistent.

Output:
- Phase clarity
- Cleanup structure issues
- Ownership concerns
- Control flow recommendations
