# spec-review

Purpose: Audit a specification for ambiguity, missing acceptance criteria, and edge cases.

Procedure:
1. Read the specification document.
2. Identify unclear or ambiguous language.
3. List missing edge cases and boundary conditions.
4. Verify acceptance criteria are measurable and testable.
5. Ensure specification contains behavior, not implementation.
6. Produce a structured critique report.

Output:
- Ambiguities
- Missing cases
- Acceptance criteria issues
- Recommendations
