# create-issue

Purpose
-------
Turn a rough idea into a structured GitHub issue.

Procedure
---------
1. Extract the core problem.
2. Write a clear issue title.
3. Draft a body with Problem, Why it matters, Constraints, Requirements, Non-goals, Acceptance criteria, and Open questions.
4. Keep the issue focused on the problem, not implementation detail.
5. Provide a `gh issue create` command using the drafted content.
6. Recommend `issue-to-spec` as the next skill.
