# design-review

Purpose
-------
Review or generate the implementation design for a spec before coding.

Procedure
---------
1. Read the relevant spec and current code structure.
2. Propose or review module boundaries, data flow, interfaces, failure handling, and test strategy.
3. Challenge unnecessary complexity.
4. Record the design in `docs/design/<issue-number>-<slug>.md`.
5. List assumptions, risks, and alternatives.

Required Output
---------------
Design summary, risks, alternatives, testing implications, and recommended approach.
