# issue-to-spec

Purpose
-------
Convert an issue or rough problem statement into a structured specification.

Procedure
---------
1. Restate the problem clearly.
2. Extract constraints, requirements, non-goals, edge cases, and acceptance criteria.
3. Write `spec/<issue-number>-<slug>.md`.
4. Keep the spec behavioral, not implementation-heavy.
5. Flag open questions explicitly.
6. Hand off to `design-review` or `implementation-plan`.

Suggested Sections
------------------
Problem, Context, Requirements, Non-goals, Edge Cases, Acceptance Criteria, Open Questions.
