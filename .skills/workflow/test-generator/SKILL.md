# test-generator

Purpose
-------
Generate or propose tests from behavior, design, code, or bug reports.

Procedure
---------
1. Identify behaviors from the spec or diff.
2. List nominal, edge, failure, and regression cases.
3. Choose the appropriate test layer: unit, integration, CLI, or BATS.
4. Map acceptance criteria to tests.
5. Prefer deterministic tests and add regression tests for bugfixes.
